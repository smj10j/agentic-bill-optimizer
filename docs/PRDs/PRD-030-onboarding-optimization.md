# PRD-030 — Onboarding Flow Optimization

**Priority**: P2
**Status**: Draft
**Registry**: [REGISTRY.md](./REGISTRY.md)

---

## Problem Statement

Time-to-value is critical. If a user doesn't see value within 2 minutes of signing up, they churn. The onboarding flow should get users to their first "wow" moment as fast as possible — seeing real insights about their money.

---

## Key Ideas

- Progressive onboarding — don't front-load permissions and setup
- Instant mock data demo before account linking so users see value immediately
- Guided first conversation with the agent to build trust
- Quick-wins identification (surface easiest savings within minutes)
- Onboarding checklist with rewards for each completed step
- A/B test onboarding variants to optimize conversion
- Skip-ahead path for power users who want to link accounts immediately

---

## Success Metrics

- Time-to-first-insight (median seconds from signup to first financial insight)
- Onboarding completion rate (% who finish all steps)
- Day-1 and Day-7 retention by onboarding variant
- Account link conversion rate (% who connect a real account)
- Drop-off rate at each onboarding step
