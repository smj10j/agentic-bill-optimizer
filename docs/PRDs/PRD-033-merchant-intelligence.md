# PRD-033 — Merchant Intelligence

**Priority**: P2
**Status**: Draft
**Registry**: [REGISTRY.md](./REGISTRY.md)

---

## Problem Statement

Users don't realize they're paying more than average at certain merchants or that cheaper alternatives exist nearby. The agent should build intelligence about merchants — pricing trends, quality signals, alternatives — to help users make informed choices.

---

## Key Ideas

- Merchant price indexing across the user base (anonymized)
- "You pay X% more than average here" insights
- Alternative merchant suggestions based on category and location
- Merchant quality scoring derived from aggregated user signals
- Loyalty program detection and value tracking
- Merchant spending trend analysis over time
- Local vs. chain comparison on price and frequency

---

## Success Metrics

- Alternative suggestion acceptance rate
- Average savings per accepted merchant switch
- Merchant intelligence coverage (% of user transactions with data)
- User engagement with merchant insights (views, clicks)
- Net spend reduction attributable to merchant switching
